# Getting Support for BeLLMark

## Before Opening an Issue

Please include the **Minimum Reproducible Bundle (MRB v1)**:

1. **Version**: BeLLMark version or git commit (`git rev-parse --short HEAD`)
2. **Export**: Relevant benchmark export (JSON format, from Results > Export)
3. **Logs**: Backend logs from the terminal where BeLLMark is running
4. **Environment**: OS, Python version, browser, Node version

## Support Channels

| Channel | Use For |
|---------|---------|
| [GitHub Issues](https://github.com/Context-Management/BeLLMark/issues) | Bugs, installation problems, feature requests |
| support@bellmark.ai | Licensing, refunds, procurement, security reports |

## Response Times

- **GitHub Issues**: Best-effort, typically within 48 hours on business days
- **Email (licensing/security)**: Within 24 hours on business days

## Tips for Fast Resolution

- Include the MRB v1 bundle (version + export + logs)
- One issue per GitHub Issue
- Search existing issues before creating a new one
- Use the appropriate issue template
